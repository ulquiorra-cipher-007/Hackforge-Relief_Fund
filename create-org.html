How It Works

This tool performs two operations in one:

  1. Creates a Supabase Auth user
    The email + password are stored in Supabase's secure auth system. Passwords are hashed — never stored in plain text.
  2. Inserts an organisations row
    The new auth user's UUID is saved as auth_user_id, linking the account to your org data permanently.
  3. Login flow
    When staff log in, the system fetches their org record by auth_user_id — the URL parameter is only a display hint and cannot be spoofed.
